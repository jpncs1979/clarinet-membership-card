# 本番デプロイ チェックリスト

本番環境へデプロイする際の確認事項です。順番に実施してください。

---

## 1. Google サービスアカウントの準備

### 1-1. Google Cloud で鍵 JSON を取得

1. [Google Cloud Console](https://console.cloud.google.com/) にログイン
2. プロジェクトを選択（または新規作成）
3. 「APIとサービス」→「ライブラリ」→ **Google Drive API** を有効化
4. 「IAM と管理」→「サービス アカウント」→ 新規作成
5. サービスアカウントの「鍵」タブ → 「鍵を追加」→ **JSON** を選択 → ダウンロード

### 1-2. サービスアカウントのメールを確認

ダウンロードした JSON 内の `client_email` を確認します（例: `xxxx@yyyy.iam.gserviceaccount.com`）。  
**このアドレスを Google Drive の共有に追加**する必要があります。

---

## 2. Google Drive の CSV を共有

1. 会員データの CSV を Google Drive にアップロード
2. CSV ファイルを右クリック → 「共有」
3. **共有相手**にサービスアカウントのメール（`client_email`）を入力
4. 権限を **閲覧者** に設定
5. 「送信」をクリック

**ファイル ID の取得：**
- 共有ダイアログの「リンクをコピー」からURLを取得
- `https://drive.google.com/file/d/【ここがファイルID】/view?...` の **`/d/` と `/view` の間** がファイルID

**現在のファイルID：** `1kj501izC4k85SUUcF5XvQXA5dOVM7q8q`

## 3. デプロイ方法ごとの設定

### 方法A: ローカル / VPS で実行する場合

#### 3A-1. 鍵ファイルの配置

1. ダウンロードした JSON 鍵を `google-service-account.json` として保存
2. **プロジェクト直下**（`server.js` と同じフォルダ）に配置

```
clarinet-membership-card/
├── server.js
├── google-service-account.json  ← ここ
├── .env
└── ...
```

#### 3A-2. .env の設定

`.env` に以下を記述：

```env
SESSION_SECRET=推測困難なランダム文字列
PORT=3000
NODE_ENV=production

GOOGLE_SERVICE_ACCOUNT_KEY_FILE=google-service-account.json
GOOGLE_DRIVE_FILE_ID=1kj501izC4k85SUUcF5XvQXA5dOVM7q8q
DRIVE_SYNC_INTERVAL_MINUTES=10
```

---

### 方法B: Render でデプロイする場合

Render はファイルシステムが一時的なため、**鍵は環境変数で渡します**。

#### 3B-1. Render の環境変数を設定

Render の Web サービス → 「**Environment**」→ 「Add Environment Variable」

| Key | Value |
|-----|-------|
| `SESSION_SECRET` | 推測困難なランダム文字列 |
| `NODE_ENV` | `production` |
| `GOOGLE_SERVICE_ACCOUNT_KEY` | 鍵 JSON の**全文**（1行で貼り付け） |
| `GOOGLE_DRIVE_FILE_ID` | `1kj501izC4k85SUUcF5XvQXA5dOVM7q8q` |
| `DRIVE_SYNC_INTERVAL_MINUTES` | `10` |

**GOOGLE_SERVICE_ACCOUNT_KEY の設定手順：**
1. ローカルの `google-service-account.json` をメモ帳で開く
2. 中身を**すべて**コピー（`{` から `}` まで）
3. Render の Value 欄に**そのまま貼り付け**（改行があってもOK）
4. `GOOGLE_SERVICE_ACCOUNT_KEY_FILE` は **設定不要**（鍵ファイルは使わない）

#### 3B-2. 補足

- `GOOGLE_SERVICE_ACCOUNT_KEY` が設定されている場合、`GOOGLE_SERVICE_ACCOUNT_KEY_FILE` は無視されます
- `lib/driveSync.js` が自動で環境変数から鍵を読み込みます

## 6. トラブルシューティング

| 現象 | 確認事項 |
|------|----------|
| 同期がスキップされる | `GOOGLE_DRIVE_FILE_ID` が設定されているか |
| `鍵ファイルが見つかりません` | ローカル: ファイルがプロジェクト直下にあるか / Render: `GOOGLE_SERVICE_ACCOUNT_KEY` を設定したか |
| `The caller does not have permission` | Drive の CSV をサービスアカウントに共有したか |
| `GOOGLE_SERVICE_ACCOUNT_KEY のJSONが無効です` | JSON のコピーが欠けていないか、余計な文字が入っていないか |
